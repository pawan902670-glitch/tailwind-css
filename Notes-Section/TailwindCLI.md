# Tailwind CSS Installation Notes

## 1. Install Tailwind CSS

### Command

```bash
npm install tailwindcss @tailwindcss/cli
```

## What does this command do?

This command downloads and installs the required Tailwind CSS packages into your project.

### `tailwindcss`

* This is the main Tailwind CSS framework.
* It contains all utility classes such as:

  * `flex`
  * `grid`
  * `text-center`
  * `bg-blue-500`
  * `p-4`

### Why is it used?

Without this package, Tailwind classes do not exist.

---

### `@tailwindcss/cli`

* CLI means **Command Line Interface**.
* It converts your Tailwind code into normal CSS that browsers understand.

### Why is it used?

Browsers cannot read Tailwind directives directly, so the CLI builds the final CSS file.

---

## What happens after installation?

The following folder and files are created:

```text
node_modules/
package.json
package-lock.json
```

### Purpose

* `node_modules` → Stores installed packages.
* `package.json` → Stores project information and dependencies.
* `package-lock.json` → Stores exact package versions.

---

# 2. Add Tailwind to Input File

## File: `src/input.css`

```css
@import "tailwindcss";
```

## What does this line do?

It imports all Tailwind CSS utilities into your project.

### Why is it used?

This line tells Tailwind:

> "Load all Tailwind classes so they can be used in HTML."

Without this line:

```html
<div class="bg-red-500"></div>
```

will not work.

---

# 3. Build Tailwind CSS

## Command

```bash
npx @tailwindcss/cli -i ./src/input.css -o ./src/output.css --watch
```

This command reads your Tailwind code and creates the final CSS file.

---

## Meaning of Each Part

### `npx`

Runs a package without installing it globally.

Example:

```bash
npx package-name
```

### Purpose

Execute installed packages directly from `node_modules`.

---

### `@tailwindcss/cli`

Runs the Tailwind compiler.

### Purpose

Converts Tailwind code into normal CSS.

---

### `-i ./src/input.css`

`-i` means **Input File**.

```text
input.css
```

This is the file containing:

```css
@import "tailwindcss";
```

### Purpose

Tells Tailwind where to read CSS from.

---

### `-o ./src/output.css`

`-o` means **Output File**.

```text
output.css
```

This file is automatically generated.

### Purpose

Stores the final CSS that browsers can understand.

---

### `--watch`

This tells Tailwind:

> "Keep watching my files and update automatically whenever I save changes."

Without `--watch`:

* You must run the command again every time you edit your HTML.

With `--watch`:

* Save file → CSS updates automatically.

---

# 4. Project Structure

```text
project-folder/
│
├── package.json
├── node_modules/
│
└── src/
    ├── index.html
    ├── input.css
    └── output.css
```

---

# 5. Connect Output CSS to HTML

Inside `index.html`:

```html
<link rel="stylesheet" href="./output.css">
```

### Why is it used?

This loads the generated CSS into the webpage.

Without this line, Tailwind styles will not appear.

---

# 6. Complete Flow

```text
Install Tailwind packages
        ↓
Create input.css
        ↓
Add:
@import "tailwindcss";
        ↓
Run build command
        ↓
Tailwind creates output.css
        ↓
Connect output.css to index.html
        ↓
Use Tailwind classes in HTML
```

---

# 7. Real Example

## input.css

```css
@import "tailwindcss";
```

## index.html

```html
<link rel="stylesheet" href="./output.css">

<h1 class="text-3xl text-blue-500 font-bold">
    Hello Tailwind
</h1>
```

## Terminal

```bash
npx @tailwindcss/cli -i ./src/input.css -o ./src/output.css --watch
```

## Result

* Tailwind reads `input.css`
* Generates `output.css`
* Browser loads `output.css`
* `text-3xl`, `text-blue-500`, and `font-bold` work correctly.

# One-Line Summary

| Item                                       | Purpose                            |
| ------------------------------------------ | ---------------------------------- |
| `npm install tailwindcss @tailwindcss/cli` | Installs Tailwind and its compiler |
| `@import "tailwindcss";`                   | Loads all Tailwind utilities       |
| `npx`                                      | Runs package commands              |
| `-i input.css`                             | Input file to read                 |
| `-o output.css`                            | Output file to generate            |
| `--watch`                                  | Automatically rebuild on changes   |
| `index.html`                               | Uses Tailwind classes              |
| `output.css`                               | Final CSS used by browser          |
